# HabitFlow

Офлайн трекер привычек. Работает как PWA — устанавливается на телефон и работает без интернета. Данные хранятся локально в браузере.

**Живая версия:** https://acronim74.github.io/habitflow/

---

## Возможности

**Привычки**
- Полезные привычки — флип карточки при отметке, время выполнения, кнопка отмены
- Вредные привычки — две кнопки: выдержал / был срыв, флип с цветом результата
- Бонусные привычки — полезная привычка вне расписания, даёт дополнительные очки
- Расписание — каждый день, будни, выходные, свои дни
- Редактирование и архивирование привычек

**Геймификация**
- Очки за каждую отметку (+10 полезная, +5 вредная, +25 за 100% дня)
- Множители за серии: ×2 от 7 дней, ×3 от 30, ×5 от 100
- 6 стадий персонажа: Начало → В ритме → Устойчивость → Сила привычки → Опора → Мастер
- 8 значков достижений
- Аватары — мужской и женский вариант для каждой стадии

**Аналитика**
- Тепловые карты активности: неделя, месяц, квартал, год
- Отдельные карты для полезных и вредных привычек
- Дневник настроения с линейным графиком (опционально)

**Темы оформления**
- ☀️ Светлая (по умолчанию)
- 🌙 Тёмная
- ◈ Трон — синий неон
- ◉ Blade Runner — янтарный неон

**Данные**
- localStorage, ключ `habitflow_data`
- Схема данных версионирована (`schemaVersion`), миграции автоматические
- Экспорт / импорт JSON через кнопки в навбаре
- Подтверждение перед перезаписью данных при импорте

## Установка как PWA

**Android (Chrome):**
1. Открыть https://acronim74.github.io/habitflow/
2. Меню (⋮) → "Добавить на главный экран"

**iPhone (Safari):**
1. Открыть https://acronim74.github.io/habitflow/
2. Кнопка "Поделиться" → "На экран Домой"

После установки приложение работает полностью офлайн.

## Структура проекта

```
habitflow/
├── index.html          — разметка и подключение скриптов
├── manifest.json       — PWA манифест
├── sw.js               — Service Worker (офлайн кэш)
├── css/
│   └── style.css       — все стили + темы ([data-theme="dark|tron|blade"])
├── js/
│   ├── assets.js       — base64 аватары и MOOD_IMAGES
│   ├── data.js         — константы: STAGES, BADGES, DEMO_HABITS, MOOD_LABELS
│   ├── state.js        — глобальное состояние, save/load, миграции схемы
│   ├── helpers.js      — утилиты: calcPoints, calcStreak, checkBadges, конфетти
│   ├── render.js       — renderToday, _buildHCard, _buildBCard, спидометр
│   ├── actions.js      — toggleCheck, markBadClean, openCreate, openEdit, импорт/экспорт
│   └── ui.js           — navigate, renderHabits/Analytics/Badges, онбординг, темы
└── icons/
    ├── icon-192.png
    └── icon-512.png
```

**Порядок загрузки скриптов:**
`assets.js` → `data.js` → `state.js` → `helpers.js` → `render.js` → `actions.js` → `ui.js`

## Локальная разработка

```bash
git clone https://github.com/Acronim74/habitflow.git
cd habitflow
npx --yes serve .
```

Открыть http://localhost:3000

Service Worker работает только на HTTPS или localhost — для локальной разработки `serve` достаточно.

---

## Обновление версии кэша

При изменении файлов Service Worker автоматически обновляет кэш при следующем открытии приложения (стратегия network-first). Ручное изменение версии не требуется.

## Структура данных

```json
{
  "schemaVersion": 1,
  "habits": [
    {
      "id": "uuid",
      "name": "Утренняя пробежка",
      "icon": "🏃",
      "category": "Спорт",
      "desc": "",
      "bad": false,
      "schedule": null,
      "checks": { "2025-03-21": true },
      "times":  { "2025-03-21": "2025-03-21T07:24:00.000Z" },
      "slips":  {},
      "clean":  {},
      "notes":  {},
      "createdAt": "2025-01-01"
    }
  ],
  "archived": [],
  "earnedBadges": ["fire"],
  "moodLog": { "2025-03-21": 3 },
  "moodEnabled": true,
  "gender": "male",
  "savedAt": "2025-03-21T10:00:00.000Z"
}
```

`schedule`: `null` = каждый день, `[0,1,2,3,4]` = будни (0=Пн, 6=Вс)  
`moodLog`: значения 0–4 (Тяжело / Так себе / Нормально / Хорошо / Отлично)
